3D RINGS

Model A — each ring's tilt precesses by a fixed step per index, azimuth divides 360° evenly.

Flat circle — a stationary flat disc fixed at dead center, facing forward. No spin, no pitch, no animation of any kind.

Webcam fill — pipes a real camera device into the flat circle. Lists physical devices on this PC (e.g. RazerCam), not OBS's internal source list — Browser Sources can't see that. Right-click this source in OBS → Interact to grant camera permission if the list stays empty.

OBS source capture — instead of the raw OS camera, pulls a live screenshot feed of a named OBS source via obs-websocket. Shows exactly what OBS is compositing (chroma key, color correction, crop, etc. all included), at the cost of choppier ~8-10fps. Requires OBS 28+ with Tools → WebSocket Server Settings → Enable WebSocket server turned on.

Internal rings — independent set that fills inward from just under the sphere's surface toward the center.

Computed automatically: 670px sphere interior ÷ ring count, so it always fills to center with no overlap.

Live Data — pulls goals, timers & hype train from your GoingLive Google Apps Script and drives ring count / percentage / color directly. Outward rings (External) = goals, regular timers & hype train. Inward rings (Internal) = spillover + GoingLive/subathon timers filling to center — same 4-step space math as the GoingLive canvas overlay, just applied to these rings.
